## 1. 产品概述

### 1.1 一句话定位

一款开源、本地优先的 **AI 原生数据库工具**，以桌面应用形态提供「可视化操作 + 自然语言对话」双模式，同时内置 **MCP Server** 能力，可被 Claude Code / Codex / OpenCode 等外部 coding agent 调用以操作数据库。

### 1.2 要解决的核心问题

- 传统 DB 工具（DBeaver/TablePlus）只解决「连接 + 查询」，没有 AI 能力；
- AI 编程助手（Claude Code 等）想操作数据库，但缺乏统一、安全的数据库接入层；
- 现有方案中「AI 看数据」的隐私边界模糊，需要明确可控的策略。

### 1.3 与同类工具的差异点

1. **双模式 AI**：对话驱动 + GUI 内 AI 辅助按钮，二者并重；
2. **MCP Server 内嵌**：工具本身既是 GUI 客户端，也是 agent 可调用的 MCP Server；
3. **环境分级权限**：dev/staging/prod 分级，prod 默认只读；
4. **Schema-only 默认**：AI 调用默认不发送实际数据行，隐私可控；
5. **开源自用**：可自托管、自定义 LLM 接入。

## 2. 已确认的核心决策

| 决策点       | 选择                                    | 说明                                        |
| ------------ | --------------------------------------- | ------------------------------------------- |
| 产品形态     | **Tauri 桌面应用**                      | 原生窗口 + Web 前端，内嵌本地后台服务       |
| AI 交互      | **双模式并重**                          | 对话 Chat + GUI 内 AI 辅助按钮              |
| Agent 集成   | **暴露 MCP Server**                     | 工具作为 MCP Server，agent 连接它操作数据库 |
| 权限安全     | **按环境分级配置**                      | dev/staging/prod，prod 默认只读             |

| AI 数据隐私  | **Schema-only 默认**                    | 默认只发 Schema，不发实际数据行             |
| 多连接       | **多连接 + 多标签页**                   | 同时管理多个连接，多 tab 切换               |

## 3. 用户角色与典型场景

### 3.1 角色

- **开发者（主要）**：日常连库、写 SQL、用 AI 辅助排查问题。

### 3.2 典型场景

| #   | 场景                                                | 涉及模块                         |
| --- | --------------------------------------------------- | -------------------------------- |
| S1  | 浏览库结构、查看表数据                              | 连接管理 + 对象浏览器 + 数据网格 |
| S2  | 用自然语言查询：「查最近 7 天下单最多的 10 个用户」 | AI 对话 + SQL 编辑器             |
| S3  | 让 AI 解释一条慢 SQL 并给优化建议                   | AI 辅助（GUI 内）+ LLM 网关      |
| S4  | 在 Claude Code 里直接问「我的 users 表有哪些字段」  | MCP Server + 连接管理            |
| S5  | 对 dev 库执行 AI 生成的 DDL，对 prod 库只预览不执行 | 权限分级 + 安全确认              |
| S6  | 同时连 MySQL 和 Redis，对比/操作                    | 多连接 + 多标签页                |

## 5. 详细功能模块

### M1. 连接管理（Connection Management）

**职责**：管理所有数据库连接配置，是所有功能的入口。

| 功能                    | 说明                                                                                               | 优先级 |
| ----------------------- | -------------------------------------------------------------------------------------------------- | ------ |
| F1.1 新建连接向导       | 选择类型（MySQL/PG/SQLite/Redis）→ 填写参数 → 测试连接 → 保存                                      | P0     |
| F1.2 连接参数           | 主机/端口/用户/密码/库名/字符集/SSL/SSH 隧道；SQLite 选文件；Redis 支持 cluster/sentinel、DB index | P0     |
| F1.3 连接测试           | 保存前 ping 一次，给出错误信息                                                                     | P0     |
| F1.4 连接分组           | 按环境（dev/staging/prod）或自定义分组；环境来自连接属性，驱动权限策略                             | P0     |
| F1.5 连接列表 UI        | 侧边栏树状展示，支持搜索、收藏、拖拽排序                                                           | P0     |
| F1.6 编辑/删除/克隆连接 | 删除需二次确认                                                                                     | P0     |
| F1.7 连接导入导出       | 导出为加密文件（不含明文密码，需主密码解密）                                                       | P1     |
| F1.8 连接健康/重连      | 显示连接状态，断线自动重连                                                                         | P1     |

## 7. 技术架构概要（模块边界，不含实现）

> 本节只界定模块职责与接口边界，不涉及具体代码。

```
[前端 UI 层]  ←→  [本地后台服务进程]  ←→  [DB / LLM / MCP 外部]
                  │
       ┌──────────┼───────────────────────┐
       │          │                       │
  连接/权限/隐私层   DB 驱动抽象层          外部接入层
  - 凭据加密    - MySQL/PG/SQLite/Redis  - LLM 网关
  - 环境分级    - 统一查询接口            - MCP Server
  - SQL 安全    - 类型映射                - Schema-only 裁剪
```

**关键边界**

- **DB 驱动层**：定义统一接口，4 种 DB 各自实现，上层不感知差异。
- **LLM 网关**：定义统一 prompt 构造 + 调用接口，多 provider 可切换。
- **权限层**：作为横切关注点，拦截所有「执行 SQL」入口（GUI / AI / MCP 三条路径都过它）。
- **MCP Server**：复用 DB 驱动层 + 权限层，是第三条执行路径。

**三条执行路径统一过权限层**（重要）：

```
GUI 用户执行 SQL  ──┐
AI 生成并执行 SQL ──┼──→ 权限层（环境分级 + 危险拦截 + 确认）──→ DB 驱动层
Agent via MCP    ──┘
```

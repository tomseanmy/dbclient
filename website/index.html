// 为什么用它

解决开发者每天
都在踩的三个坑。

围绕真实的数据库工作方式打造，而非花哨的演示。

痛点 / 复杂查询

手写长 SQL 太累

用自然语言描述意图，AI 直接生成 SQL。NL2SQL + 流式对话 + Agent 多步执行，生成的写操作 SQL 落入编辑器，由你确认后再执行。

痛点 / 跨库搬迁

迁移脚本容易写错

结构 diff + 数据 diff + 跨库，方言感知自动生成 DDL/DML。4 步向导 + 风险徽章，计划可保存、脚本可导出。

痛点 / 误操作

怕在生产库上闯祸

环境分级权限，prod 默认只读。DROP / TRUNCATE / 缺 WHERE 的写操作在触达数据库前被拦截，支持临时提权并全程审计。

// 核心能力

六大能力，全部开箱即用。

每一项都已实现并在日常使用，而非路线图上的承诺。

AI 对话 + 流式 + Agent

NL2SQL、流式响应、Agent tool-calling 多步自动执行。

  • OpenAI 兼容：DeepSeek / Qwen / Moonshot / Ollama
  • 编辑器右键「解释 / 优化」
  • 对话历史持久化 · token 用量记录

数据库迁移

结构 diff + 数据 diff + 跨库，方言感知 DDL/DML。

  • 4 步向导 · 风险徽章 safe/caution/danger
  • 计划保存复用 · 脚本一键导出
  • MySQL / PostgreSQL / SQLite

默认只发 Schema

AI 看到表 / 列 / 类型 / 注释，永远不是真实数据行。

  • 裸用驱动、不上 ORM
  • 样本数据发送 / PII 脱敏（规划中）

环境分级权限

dev / staging / prod 分级，prod 默认只读。

  • DROP / TRUNCATE / 缺 WHERE 拦截
  • 30 分钟临时提权 · 全过程审计

表结构编辑器

列 / 索引 / 外键行内编辑。

  • 实时预览生成的 ALTER 语句
  • 按方言差异化输出

多数据库原生支持

MySQL / PostgreSQL / SQLite / Redis。

  • 原生 SQL，无 ORM
  • Redis single / cluster
  • 多连接 · 多 tab 切换
// 安全架构

一道权限闸门，两条执行路径。

GUI 点击与 AI 对话 / Agent 调用全部流经同一道闸门。AI 执行 SQL 与你亲自操作，安全等级完全一致。

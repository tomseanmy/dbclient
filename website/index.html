// 01 — 核心能力

真正理解「意图」的数据库工具，
而不仅仅是「语法」解析。

围绕开发者当下真实的数据库使用方式打造：手写 SQL、问 AI、让 Agent 代劳。 四大能力，缺一不可。

AI 双模式

既能用自然语言和数据库对话，也能继续手写 SQL —— 两种方式都是一等公民。 对话面板和 GUI 任意位置都能唤起 AI。但 AI 永远不擅自动手：执行前必须由你确认。

  • 一键调用 describe_table
  • SQL 编辑器右键即可「解释 / 优化」
  • 生成的 SQL 落入编辑器供你审阅

内置 MCP Server

这款应用本身就是一个 MCP Server。把它接入 Claude Code、Codex 或 OpenCode， 让 coding agent 直接查 Schema、读表、执行只读 SQL —— 所有操作都走和 GUI 完全相同的权限层。 

// claude_code_config.json
{
  "mcpServers": {
    "ai-db": {
      "command": "ai-db-client",
      "args": ["mcp", "stdio"]
    }
  }
}

默认只发 Schema

AI 看到的是表 / 列 / 类型 / 注释 —— 永远不是真实的数据行。 这条边界清晰可配。想要更聪明的回答？可以按会话临时开启「发送前 N 行样本」， 但默认是关闭的。

  • 默认：只发 Schema（表结构、列、类型、注释）
  • 每个会话可单独开启样本数据
  • 列级 PII 脱敏（正则规则 · 规划中）

环境分级权限

给每个连接打上 dev / staging / prod 标签。 生产环境默认只读。DROP、TRUNCATE、缺 WHERE 的写操作，统统在触达数据库之前被拦截 —— 每条路径、每次调用都不例外。

dev
RWDDL
staging
RWDDL
prod
RWDDL

W = 写，DDL = 改表结构。黄色 = 弹窗确认， 红色 = 直接拦截。生产可临时提权，全过程留痕。

// 02 — 架构

一道 权限闸门，
三条执行路径。

核心设计：GUI 点击、AI 对话、MCP agent 调用 —— 全部流经同一道闸门。 也就是说，agent 经 MCP 操作和你亲自在 GUI 操作，绝对不可能存在不同的安全等级。 它们共享一道。

// 04 — 技术栈

朴素、现代、本地优先。

桌面应用，渲染层是 React，后台是 Node，整条数据通路用 TypeScript strict 端到端类型化。

外壳
Electron 42 + electron-vite
原生窗口 + Node 后台，跨平台打包开箱即用。
渲染层
React 18 · Vite · Zustand
快速 HMR；Monaco Editor + TanStack Table 处理 SQL 与数据网格。
驱动
mysql2 · pg · better-sqlite3 · ioredis
裸用，不上 ORM。每条 SQL 都归我们自己掌控。Redis 支持 single / cluster / sentinel。
SQL 解析
node-sql-parser · sql-formatter
AST 解析做安全分析，格式化输出友好可读。
AI
OpenAI 兼容 LLM 网关
DeepSeek · Qwen · Moonshot · Ollama，Schema-only 默认上下文，token 用量记录。
MCP
@modelcontextprotocol/sdk
官方 TS SDK，stdio / SSE 传输，复用同一道权限层。
凭据
keytar（macOS Keychain / Win Vault）
密码 / API Key 永远不以明文落盘。
架构
核心领域层零 Electron 依赖
GUI / AI / MCP 三条执行路径共享同一套业务逻辑与权限闸门。
